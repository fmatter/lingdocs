Introduction

This document serves both as a test for pylingdocs and a showcase of its capabilities. It aims to contain every feature and model currently available in pylingdocs. The pylingdocs-flavored markdown on which is it based can be found here. It uses data from the CLDF test dataset.

Integrating linguistic entities

The core functionality of pylingdocs is including linguistic data in your prose. This is achieved by using the cldfviz tool to access datasets in the CLDF format. A number of models are built-in, at the moment:

  1. morphemes: -se
  2. morphs: -je
  3. languages: Tiriyó
  4. texts: “Ekïrï”

You can also use cldfviz syntax: Apalaí

You can also refer to multiple entities of the same kind: The Tiriyó suffix -se has the allomorphs -se, -je, and -e. Apalaí, and Tiriyó are both Cariban languages, as are Ikpeng, Upper Xingu Carib, and Pemón.

Examples

Interlinearized glossed examples

Glossed examples are called like any other entity, but are rendered as follows:

  1. Ikpeng
    ekïrï ru man eroang tïmamin ïna
    ekɨrɨ old
    ru part
    man aff
    ero-aŋ go-rem .cont
    tɨ-mami-n cor -work-pert
    ɨna obl
    ‘The old man went to his work,’
    1. Ikpeng
      yuwyang man tïwïn:
      j-uwj-aŋ 3 >3 -search-rem .cont
      man aff
      tɨ-wɨ-n cor -machete-pert
      ‘He looked for his machete.’
    2. Ikpeng
      ïwïn mun eto ankang man
      ɨ-wɨ-n 1 poss -machete-pert
      mun inan .dist
      eto uncert
      an-k-aŋ 3 -say-rem .cont
      man aff
      ‘“Where might my machete be?”, he asked.’

Manual examples

These can contain tables, lists, whatever:

  1. Tiriyó -se
    Apalaí -se

They can also have multiple parts:

    1. Tiriyó -se
      Apalaí -se
    2. -se

Arguments

Pass arguments to the visualizer:

  1. Ikpeng
    rïk rïk rïk yuwitkang man
    rɨk ideo .hear
    rɨk ideo .hear
    rɨk ideo .hear
    j-uwi-tk-aŋ 3 >3 -search-iter -rem .cont
    man aff
    ‘So he went looking for it.’
